What Is Workflow Copilot?

Learn how Workflow Copilot helps you create automations by describing what you want in plain language.

Simple explanation
  • Traditional workflow building: you manually choose triggers, actions, connectors, and conditions.
  • Workflow Copilot: you describe the workflow you want, and Copilot drafts the automation steps for you.
  • Your role: review the generated steps, check the logic, test the workflow, and adjust it if needed.
Key building blocks
  • Trigger: when the workflow starts, such as a schedule or new email.
  • Action: what happens next, such as getting calendar events or sending a Teams message.
  • Output: the useful result, such as a reminder, digest, saved file, or notification.
Copilot can help build the flow, but you still need to check whether the logic, data access, and output are correct.

Demo: Today’s Calendar Digest to Teams

Build a workflow that runs every weekday morning, gets today’s remaining calendar events, and sends a digest to Teams.

Workflow goal
Every weekday morning, get today’s upcoming calendar events and send a simple digest to Teams.
Step 1: Trigger
  • Create a scheduled workflow.
  • Set it to run every weekday morning.
  • Use Singapore time zone if prompted.
Weekday Morning Recurrence Run every weekday morning at 8:30 AM Singapore time.
Step 2: Get today’s calendar events

Use M365 Copilot or calendar-related action to retrieve today’s events.

Find all of my calendar events scheduled for today. For each event, return only the title, start time, and end time. Do not include meeting links, join URLs, or any online meeting data. Only include events that have not yet ended as of the time this workflow runs. User's timezone: Asia/Singapore.
Step 3: Send digest to Teams

Send the output to Microsoft Teams using Chat with Flow bot or your preferred Teams destination.

Send a Teams message in Chat with Flow bot to me. The message should summarize today's upcoming calendar events from Get today's calendar events, listing each event's title, start time, and end time in Singapore time. Do not include meeting links or URLs. If there are no events, state: You have no upcoming events today. Format the message in clear, plain text with each event on a new line.
This demo shows how one workflow can combine schedule, Microsoft 365 context, and Teams notification.

Build Your Own Workflow

Pick one simple workflow idea and use Copilot to generate the first version. Then review, test, and refine it.

How to start
  • Choose one repetitive task you do often.
  • Describe the trigger: when should the workflow start?
  • Describe the action: what should happen next?
  • Describe the output: where should the result go?
  • Review the generated workflow before enabling it.
Starter prompt
Create a workflow that runs every weekday morning and sends me a Teams message with a short reminder of what I should check today.
Beginner workflow ideas
  • Daily calendar digest: send today’s remaining meetings to Teams.
  • Email attachment reminder: notify me in Teams when an email arrives with an attachment.
  • Weekly planning reminder: every Monday morning, send a reminder to review priorities.
  • Meeting preparation reminder: before customer meetings, remind me to review notes and prepare questions.
  • End-of-day wrap-up: every weekday evening, ask me to record completed tasks and blockers.
Start small. A simple workflow that works is better than a complex workflow that nobody trusts.

Wrap-up

Best Practices & Common Mistakes

Learn how to review a Copilot-generated workflow before trusting it in real work.

Best practices
  • Start with one clear task.
  • Use specific instructions and clear step names.
  • Test with your own account before sending to a group.
  • Check whether the workflow includes unnecessary sensitive data.
  • Review the run history after testing.
Common mistakes
  • The trigger is too broad and runs too often.
  • The Teams message includes too much detail.
  • The workflow sends to the wrong person or channel.
  • The user enables the workflow without testing it first.
Workflow Copilot helps you build faster, but responsible review is still part of the job.
